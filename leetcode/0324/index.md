# 0324：摆动排序 II（★★★）


## 题目

给你一个整数数组 nums，将它重新排列成 nums[0] < nums[1] > nums[2] < nums[3]... 的顺序。

你可以假设所有输入数组都可以得到满足题目要求的结果。

提示：

- 1 <= nums.length <= 5 * 10^4
- 0 <= nums[i] <= 5000
- 题目数据保证，对于给定的输入 nums ，总能产生满足题目要求的结果
 
进阶：你能用 O(n) 时间复杂度和 / 或原地 O(1) 额外空间来实现吗？


<!--more--> 


示例 1：

    输入：nums = [1,5,1,1,6,4]
    输出：[1,6,1,5,1,4]
    解释：[1,4,1,5,1,6] 同样是符合题目要求的结果，可以被判题程序接受。

示例 2：

    输入：nums = [1,3,2,2,3,1]
    输出：[2,3,1,3,1,2]
     


## 分析

容易想到将较小的一半数放在偶数位上，较大的一半数放在奇数位上。证明也比较容易：

    假设一个可行方案存在一个较大的数在偶数位上，那么必然还存在一个较小的数在奇数位上
    将这两个数交换位置，显然依然是一个可行方案
    故只要有解，必然有一个解满足：较小的一半数都在偶数位上，较大的一半数都在奇数位上

令 a = max(较小的一半数)，b = min(较大的一半)。如果 a < b，那么就已经解决了。

当 a==b 时，需要尽量让 a 和 b 不相邻。要么让 a 靠右、b 靠左，要么让 a 靠左、b 靠右。
考虑到 a 和 b 也可能出现多次，所以要么较小的一半数和较大的一半数都顺序排列，要么都逆序排列。

试验中发现逆序排列更优，比如用例 [1，2，4，4，4，6] 只能用逆序排列而不能用顺序。
逆序排列也相当于从大到小先依次放奇数位置，再依次放偶数位置。

最后证明一下该方案不可行时，必然无解。

    假设 n=len(nums) 为偶数，该方案不可行时意味着某个数 x 出现了至少 n//2 + 1 次，
    显然不管怎么排列都有两个 x 相邻，无解。
    
    假设 n 为奇数，该方案不可行时意味着某个奇数位上的数 x 出现了至少 (n+1)//2 次，
    并且记该方案中末尾数为 y，有 y <= x。
    为了使 x 都不相邻，只能将 x 都放在偶数位上。但此时 y 无处可放，无解。
    

## 解答

```python
def wiggleSort(self, nums: List[int]) -> None:
    nums.sort(reverse=True)
    nums[1::2], nums[::2] = nums[:len(nums)//2], nums[len(nums)//2:]
```
