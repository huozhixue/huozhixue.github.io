# 0091：解码方法（★★）


## 题目

一条包含字母 A-Z 的消息通过以下映射进行了 编码 ：

	'A' -> 1
	'B' -> 2
	...
	'Z' -> 26
	
要 解码 已编码的消息，所有数字必须基于上述映射的方法，反向映射回字母（可能有多种方法）。例如，"111" 可以将 "1" 中的每个 "1" 映射为 "A" ，从而得到 "AAA" ，或者可以将 "11" 和 "1"（分别为 "K" 和 "A" ）映射为 "KA" 。注意，"06" 不能映射为 "F" ，因为 "6" 和 "06" 不同。

给你一个只含数字的 非空 字符串 num ，请计算并返回 解码 方法的 总数 。

题目数据保证答案肯定是一个 32 位 的整数。


<!--more--> 

示例 1：

	输入：s = "12"
	输出：2
	解释：它可以解码为 "AB"（1 2）或者 "L"（12）。

示例 2：

	输入：s = "226"
	输出：3
	解释：它可以解码为 "BZ" (2 26), "VF" (22 6), 或者 "BBF" (2 2 6) 。

示例 3：

	输入：s = "0"
	输出：0
	解释：没有字符映射到以 0 开头的数字。含有 0 的有效映射是 'J' -> "10" 和 'T'-> "20" 。由于没有字符，因此没有有效的方法对此进行解码，因为所有数字都需要映射。

示例 4：

	输入：s = "06"
	输出：0
	解释："06" 不能映射到 "F" ，因为字符串开头的 0 无法指向一个有效的字符。 


## 分析

### #1

头部解码可以分为两类：

- 第一个字符映射，剩下的是递归子问题
- 前两个字符映射，剩下的是递归子问题

注意第一个字符或前两个字符不一定能映射，需要判断。

```python
@lru_cache(None)
def numDecodings(self, s: str) -> int:
	if s[0] == '0':
		return 0
	if len(s) == 1:
		return 1
	res = self.numDecodings(s[1:])
	if int(s[:2]) <= 26:
		res += self.numDecodings(s[2:]) if len(s) > 2 else 1
	return res
```

44 ms

### #2 

可以改写为考虑动态规划。为了方便，令 dp[i] 代表 s[:i+1] 的解码总数，则状态转移方程为：
	
	if i==0:	dp[0] = int(s[i]!='0')
	elif i==1:	dp[i] = int(s[i]!='0')*dp[i-1] + int(10<=int(s[i-1:i+1])<=26)
	else:		dp[i] = int(s[i]!='0')*dp[i-1] + int(10<=int(s[i-1:i+1])<=26)*dp[i-2]

```python
def numDecodings(self, s: str) -> int:
	n = len(s)
	dp = [0] * n
	for i in range(n):
		if i==0:
			dp[i] = int(s[i]!='0')
		elif i==1:
			dp[i] = (s[i]!='0')*dp[i-1] + int(10<=int(s[i-1:i+1])<=26)
		else:
			dp[i] = (s[i]!='0')*dp[i-1] + (10<=int(s[i-1:i+1])<=26)*dp[i-2]
	return dp[-1]
```

36 ms

### #3

递推过程只和前两个状态有关，所以可以只用两个变量保存

## 解答

```python
def numDecodings(self, s: str) -> int:
	a, b = 1, int(s[0]!='0')
	for i in range(1, len(s)):
		a, b = b, (s[i]!='0')*b + (10<=int(s[i-1:i+1])<=26)*a
	return b
```

32 ms
